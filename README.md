# “学术之光”科研资源智能共享平台构想

## 1. 项目愿景
- 打造一个集资源共享、智能协作与科研指导于一体的综合平台。
- 通过智能化推荐和协作工具降低科研门槛，提升师生科研效率。
- 营造开放、可信赖的学术社区，激发学生科研兴趣并沉淀科研成果。

## 2. 目标用户与角色
1. **本科生/研究生**：获取科研资源、寻找导师、参与课题实践。
2. **导师/教师**：发布课题、指导学生、管理团队协作。
3. **科研团队/社团**：共享资料、分工协作、展示成果。
4. **管理员**：维护平台运营、审核资源与用户身份、监控系统健康。

## 3. 核心模块与功能
| 模块 | 功能扩展描述 |
| --- | --- |
| 竞赛情报与战队组建 | 采集并可视化近五年ACM、数学建模、互联网+、挑战杯等赛事获奖数据，支持按院系/战队/导师多维筛选；发布赛事公告、校内选拔、备赛日历；“找队友”看板可发起招募、快速匹配导师/教练，并在协作空间内沉淀复盘资料。 |
| 科研项目与协作 | 展示校内近期科研成果（论文、专利、开放课题）及其影响力热度；提供科研方向订阅、导师课题招募、实验室开放日信息；内置招募表单与团队空间，支持任务拆解、数据集共享、里程碑看板。 |
| 保研经验共享社区 | 汇总历年保研数据（推免比例、夏令营录取、目标院校名单）并生成趋势图；学长学姐可上传经验帖、时间轴、材料模板、面试题回顾，支持直播/音视频分享与问答互动；AI 助理可据个人画像给出备保策略。 |
| 资源共享中心 | 统一托管软件安装包、镜像、环境配置脚本、实验数据；版本/依赖信息自动生成，支持秒传校内网盘、镜像源加速；提供权限控制、下载统计与违规举报，保障资源合规使用。 |
| 校园办公自动化 | 与教务、学工、图书、后勤等系统对接，实时获取校内公告、会议通知、场地预约状态；提供一站式审批进度、课表/讲座日程上墙、待办清单提醒，并可推送至移动端或企业微信。 |

## 4. 进阶功能设想
1. **竞赛&科研知识图谱**：将赛事获奖、课题方向、导师团队、资源库等信息串联，帮助学生发现潜在合作与备赛/科研路径。
2. **AI 学术与备赛助理**：提供竞赛题目解读、科研文献初筛、保研材料润色建议，自动生成训练/学习计划。
3. **项目与赛事管理工作台**：统一支持科研课题、竞赛备战、保研材料的任务分配、预算申请、审批流追踪与成果归档。
4. **资源激励与信誉体系**：上传高质量资源、分享经验、指导新生可获得积分与徽章，用于兑换实验室参观、竞赛训练营名额等。
5. **合规与数据治理中心**：覆盖软件版权校验、数据脱敏、导师审批流及访问审计，保障资源与通知发布可追溯。
6. **多端协同体验**：Web + 小程序 + IM Bot，支持动态推送、课堂/会议签到、移动端审批。
7. **开放 API 与消息总线**：提供赛事信息接口、审批状态 Webhook、日程订阅 feed，方便与校园门户、迎新系统等集成。

## 5. 技术实现要点（课程项目友好版）
- **整体架构**：以前后端分离的单体应用为主。前端使用 Vite + Vue 3 单页应用，后端使用 Node.js（Express/Koa）提供简单 REST API，数据库选 MySQL 或 SQLite，方便本地开发与部署。
- **推荐/搜索能力**：先用关键词匹配与标签筛选（MySQL LIKE 或全文索引）提供“相关推荐”，后续再考虑引入向量检索。
- **数据采集与可视化**：优先支持人工录入与 Excel/CSV 批量导入；需要自动化时，使用 Python requests + BeautifulSoup 或 Node.js axios + cheerio 写最小化脚本，配合 node-cron 定时更新即可。
- **实时协作/通知**：大部分互动使用轮询（定时刷新）或简单的 Socket.IO 推送，避免上来就做复杂的 WebRTC。
- **安全与权限**：基于 JWT 的登录态 + 简单 RBAC（学生/导师/管理员三角色）；密码加密保存，接口层做基础的参数校验与速率限制。
- **日志与部署**：开发期用 nodemon + console.log，部署时配合 Winston/PM2 日志轮转；单台服务器或校内机房即可，用 Nginx 反代 + PM2 常驻，满足课程展示。

## 6. 推荐技术栈（简化方案）
1. **前端层**：Vite + Vue 3 + JavaScript（可选 TypeScript），组件库选 Element Plus；状态用 Pinia，网络请求用 Axios，样式使用 UnoCSS/Tailwind 任选其一即可。
2. **移动/小程序**：优先做响应式 H5，必要时用 Uni-App 复用 Vue 代码输出微信小程序，无需额外框架。
3. **后端层**：Express + TypeScript/JavaScript，配合 Sequelize/Prisma（二选一）访问 MySQL/SQLite；若需要实时能力，可直接集成 Socket.IO。
4. **数据与存储**：MySQL 5.7/8 作为主库；文件/图片可暂存在服务器本地或阿里云 OSS/七牛云学生套餐，方便演示。
5. **任务与消息**：Node-cron + 数据库表记录任务状态；消息通知直接写入数据库并在前端轮询刷新，不引入消息队列。
6. **部署与工具链**：Node.js LTS + npm scripts；使用 PM2 或 forever 常驻进程，Nginx 负责静态资源与反向代理；CI 可通过 GitHub Actions 简单跑 lint/build 即可。

## 6. 用户旅程示例
1. 学生登录后浏览竞赛仪表盘 → 根据获奖榜样与时间轴选择赛事 → 在“找战队”中发布招募需求并拉起协作空间。
2. 同时订阅科研方向 → 收到导师课题招募 → AI 助理生成阅读清单与任务计划，团队在看板上跟踪进度。
3. 备保阶段查看经验库 → 预约学长线上分享 → 上传材料请导师批注 → 校园 OA 自动推送结果。
4. 所需软件与数据在资源中心一键获取 → 办公自动化模块同步教务通知和重要会议提醒，确保信息不滞后。
